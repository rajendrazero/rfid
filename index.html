Tujuan & Requirements

Tujuan Aplikasi

Membangun sistem manajemen troli bandara berbasis RFID untuk meningkatkan efisiensi pelacakan dan manajemen troli di lingkungan bandara.

  • Mengurangi waktu pencarian troli oleh petugas
  • Memantau ketersediaan troli di berbagai lokasi bandara
  • Memberikan data real-time tentang status troli
  • Meningkatkan pengalaman pengguna dengan troli yang mudah ditemukan
  • Memudahkan manajemen inventaris troli

Functional Requirements

  1. Sistem login dengan role user dan admin
  2. Pelacakan lokasi troli melalui RFID
  3. Fitur CRUD (Create, Read, Update, Delete) untuk troli
  4. Pemberian status troli (tersedia, rusak, dalam perbaikan)
  5. Monitoring user aktif dan aktivitas mereka
  6. Dashboard informasi troli dengan detail lengkap
  7. Laporan historis pergerakan troli

Endpoint Troli

GET /trolleys

Ambil semua troli (dengan filter, pencarian, pagination, dan informasi lokasi untuk visualisasi)

Requires Authentication: Bearer Token

Parameters

location_id (optional, UUID) - Filter berdasarkan lokasi troli saat ini
status_id (optional, UUID) - Filter berdasarkan status troli
search (optional, string) - Pencarian berdasarkan kode troli atau RFID
limit (optional, integer, default: 20, min: 20) - Jumlah data per halaman
page (optional, integer, default: 1) - Nomor halaman

Response (Success 200)

                "message": "Success",

                        "id": "uuid",
                        "rfid_code": "RF123456",

                        "id": "uuid",
                        "name": "Aktif"
                        },
                        "location": {
                        "id": "uuid",
                        "name": "Terminal 1",
                        "x_coordinate": 120.5,
                        "y_coordinate": 310.2
                        }
                    },
                    {
                        "id": "uuid",
                        "rfid_code": "RF654321",
                        "trolley_code": "TR-02",
                        "status": {
                        "id": "uuid",
                        "name": "Tersedia"
                        },
                        "location": {
                        "id": "uuid",
                        "name": "Terminal 2",

Response (Error 400)

                }

Terjadi ketika parameter query tidak valid

{
                "status": false,
                "message": "Tidak ada troli yang cocok dengan filter pencarian",
                "data": [],
                "error": 404
                }

Terjadi ketika tidak ada troli yang ditemukan berdasarkan filter

Contoh Penggunaan

Mencari troli dengan lokasi tertentu, status tertentu, kode TR-01, menampilkan 10 item per halaman di halaman ke-2

Informasi Visualisasi Lokasi

Koordinat lokasi (x_coordinate dan y_coordinate) dapat digunakan untuk visualisasi peta lokasi troli di aplikasi:

  • Menampilkan posisi troli pada peta bandara
  • Visualisasi kepadatan troli di area tertentu
  • Heatmap distribusi troli
  • Rute pergerakan troli historis

Potensi Kesalahan

  • Menggunakan format UUID tidak valid untuk location_id atau status_id
  • Menggunakan nilai di luar range 20-1000 untuk limit
  • Menggunakan nilai negatif untuk page
  • Filter yang terlalu spesifik sehingga tidak ada troli yang cocok

Penggunaan Sistem Pelacakan

Endpoint ini merupakan bagian penting dari sistem pelacakan troli berbasis RFID:

  • Setiap kali troli terdeteksi oleh RFID reader, endpoint ini dipanggil
  • Koordinat offset membantu menentukan posisi tepat troli dalam area
  • Data yang dikumpulkan digunakan untuk visualisasi peta real-time
  • Historis pergerakan troli dapat dianalisis untuk efisiensi operasional

Best Practices

  • Gunakan sensor RFID di titik-titik strategis di seluruh bandara
  • Pastikan setiap pembacaan RFID memicu panggilan ke endpoint ini
  • Gunakan nilai offset untuk akurasi posisi yang lebih baik
  • Catat informasi "detected_by" untuk pelacakan sumber deteksi
  • Monitor error 500 untuk mendeteksi masalah server

Script Pembuatan Tabel

CREATE TABLE public.locations (
    id uuid NOT NULL DEFAULT gen_random_uuid(),
    name character varying NOT NULL,
    description text,
    created_at timestamp with time zone DEFAULT now(),
    x_coordinate numeric,
    y_coordinate numeric,
    CONSTRAINT locations_pkey PRIMARY KEY (id)
);

CREATE TABLE public.roles (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  name character varying NOT NULL UNIQUE,
  CONSTRAINT roles_pkey PRIMARY KEY (id)
);

CREATE TABLE public.trolley_location_logs (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  trolley_id uuid NOT NULL,
  location_id uuid NOT NULL,
  detected_at timestamp with time zone NOT NULL DEFAULT now(),
  detected_by uuid,
  x_offset numeric,
  y_offset numeric,
  CONSTRAINT trolley_location_logs_pkey PRIMARY KEY (id),
  CONSTRAINT trolley_location_logs_detected_by_fkey FOREIGN KEY (detected_by) REFERENCES public.users(id),
  CONSTRAINT trolley_location_logs_location_id_fkey FOREIGN KEY (location_id) REFERENCES public.locations(id),
  CONSTRAINT trolley_location_logs_trolley_id_fkey FOREIGN KEY (trolley_id) REFERENCES public.trolleys(id)
);

CREATE TABLE public.trolley_statuses (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  name character varying NOT NULL,
  description text,
  CONSTRAINT trolley_statuses_pkey PRIMARY KEY (id)
);

CREATE TABLE public.trolleys (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  rfid_code character varying NOT NULL UNIQUE,
  trolley_code character varying NOT NULL UNIQUE,
  status_id uuid,
  created_at timestamp with time zone DEFAULT now(),
  updated_at timestamp without time zone,
  deleted_at timestamp without time zone,
  current_location_id uuid,
  CONSTRAINT trolleys_pkey PRIMARY KEY (id),
  CONSTRAINT trolleys_status_id_fkey FOREIGN KEY (status_id) REFERENCES public.trolley_statuses(id),
  CONSTRAINT trolleys_current_location_id_fkey FOREIGN KEY (current_location_id) REFERENCES public.locations(id)
);

CREATE TABLE public.users (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  username character varying NOT NULL UNIQUE,
  password text NOT NULL,
  fullname character varying,
  email character varying UNIQUE,
  role_id uuid,
  created_at timestamp with time zone NOT NULL DEFAULT now(),
  updated_at timestamp without time zone,
  deleted_at timestamp without time zone,
  is_logged_in boolean DEFAULT false,
  last_login timestamp with time zone,
  CONSTRAINT users_pkey PRIMARY KEY (id),
  CONSTRAINT users_role_id_fkey FOREIGN KEY (role_id) REFERENCES public.roles(id)
);

Tahapan Pembuatan API

1

Setup Autentikasi

Fundamental

Implementasi sistem keamanan dan manajemen pengguna sebagai dasar semua fungsi API.

Endpoint Prioritas

  • POST /auth/register - Pendaftaran user/admin
  • POST /auth/login - Login dengan JWT
  • POST /auth/refresh - Refresh token
  • POST /auth/logout - Terminate session

Workflow Implementasi

  1. Setup database user
  2. Implementasi JWT authentication
  3. Role-based access control
  4. Token refresh mechanism

Best Practice

Gunakan middleware otentikasi untuk melindungi endpoint sensitif. Simpan refresh token di database untuk validasi.

2

Manajemen Troli

Core Business Logic

Membangun sistem CRUD troli dasar sebagai inti aplikasi manajemen troli.

Endpoint Prioritas

  • POST /trolleys - Buat troli baru
  • GET /trolleys - Ambil daftar troli (filter/pagination)
  • GET /trolleys/{id} - Detail troli spesifik
  • PUT /trolleys/{id} - Update data troli
  • DELETE /trolleys/{id} - Hapus troli

Struktur Data 

Troli {
            id: UUID
            rfid_code: String
            trolley_code: String
            status_id: UUID (FK)
            created_at: DateTime
            updated_at: DateTime
        }

Penting

Implementasikan validasi RFID unik untuk mencegah duplikasi. Gunakan soft delete untuk riwayat historis.

3

Pelacakan Lokasi

Core RFID Functionality

Implementasi sistem pelacakan berbasis RFID untuk memonitor pergerakan troli secara real-time.

Endpoint Prioritas

  • POST /trolleys/{id}/location-logs - Catat lokasi baru
  • GET /trolleys/{id}/logs - Riwayat lokasi troli
  • GET /trolleys/location/{rfid} - Lokasi terkini via RFID

Workflow RFID

  1. Troli terdeteksi oleh RFID reader
  2. Kirim lokasi ke endpoint API
  3. Update database dengan lokasi baru
  4. Tampilkan di peta real-time

Integrasi Hardware

Pastikan RFID reader dikonfigurasi untuk POST request ke endpoint lokasi-logs setiap kali troli terdeteksi. Gunakan koordinat offset untuk akurasi posisi.

4

Manajemen Status & Lokasi

Metadata Management

Sistem pendukung untuk operasional troli berupa manajemen status dan lokasi bandara.

Endpoint Prioritas

  • GET /trolley-statuses - Daftar status troli
  • POST /trolley-statuses - Buat status baru (admin)
  • GET /locations - Daftar lokasi bandara

Struktur Metadata 

Status {
            id: UUID
            name: String (Tersedia, Digunakan, Maintenance)
            description: String
        }
        
        Location {
            id: UUID
            name: String
            x_coordinate: Float
            y_coordinate: Float
        }

Penggunaan

Status dan lokasi digunakan untuk filtering di dashboard utama. Koordinat lokasi digunakan untuk visualisasi peta.

5

Sistem Monitoring

Admin Dashboard

Fitur pengawasan untuk administrator memonitor aktivitas sistem dan pengguna.

Endpoint Prioritas

  • GET /users/monitoring - Aktivitas user & akses troli

Data yang Dimonitor

  • User aktif (last login)
  • Aktivitas terakhir user
  • Statistik akses troli
  • Top user pengakses troli

Keamanan

Hanya admin yang bisa mengakses endpoint ini. Audit log harus mencatat semua akses ke data monitoring.

6

Manajemen Pengguna Lanjut

Admin Controls

Fitur administratif untuk manajemen pengguna dan kontrol akses sistem.

Endpoint Prioritas

  • PATCH /users/{id}/role - Ubah peran user (admin)

Kontrol Akses

  • Admin bisa ubah role user lain
  • Admin tidak bisa ubah role sendiri
  • Verifikasi ganda untuk perubahan role

Audit Log

Catat semua perubahan role dengan informasi: user yang diubah, role baru, admin yang mengubah, dan timestamp.
